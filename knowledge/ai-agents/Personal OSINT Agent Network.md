# Personal OSINT Agent Network

## Core Principle

As social feeds become mixed human-agent environments, the default reader needs a defensive intelligence layer. A personal OSINT agent network is a curated system for checking claims, tracing provenance, watching trusted sources, detecting manipulation, and protecting attention before reacting to the feed.

The goal is not to outsource judgment. The goal is to give judgment better inputs.

## What The Network Should Do

- Track a trusted source map by domain, topic, and source quality.
- Check factual claims against primary sources or high-quality secondary sources.
- Preserve provenance: original post, author, timestamp, linked evidence, edits, and repost chains.
- Detect common manipulation patterns such as duplicated language, coordinated posting, rage bait, fake consensus, and source laundering.
- Compare claims across ideological, geographic, and institutional vantage points.
- Maintain personal filters for accounts, topics, and feed patterns that reliably damage attention or mood.
- Save useful verified claims back into the wiki with source pages rather than leaving them in the feed.

## Happiness And Attention Layer

The defensive layer should not only ask "is this true?" It should also ask:

- What does repeatedly consuming this do to attention?
- Does this source create useful situational awareness or addictive agitation?
- Is this feed pattern making agency clearer or weaker?
- Which topics deserve monitoring, and which should be quarantined unless they affect a live decision?

This is the "OSINT map for happiness" angle: a personal intelligence network should protect both epistemics and emotional operating conditions.

## Architecture Notes

- Keep private context local by default.
- Use narrow tools for browsing, extraction, citation capture, and source comparison.
- Treat social posts and web pages as untrusted input.
- Require human review before posting, messaging, or publicly amplifying anything.
- Log the evidence trail so conclusions can be replayed later.
- Prefer recurring dossiers and durable wiki pages over ephemeral chat conclusions.
