# Minimalist Entrepreneur Skills

## What This Source Is

A public GitHub repository that packages Sahil Lavingia's minimalist-entrepreneur framework as Claude-oriented agent skills. The repo is mostly plugin metadata plus ten markdown skill files.

## Key Ideas

- A skill library can encode a business operating model, not just technical documentation.
- Markdown skills can turn recurring founder questions into reusable decision prompts around community, idea validation, MVP scope, manual delivery, first customers, pricing, marketing, growth, values, and review.
- The manual -> processized -> productized progression appears to be the core operating pattern behind the set.

## Claims Worth Tracking

- External skills can be valuable even when their main content is business methodology rather than technical know-how.
- Some knowledge is most useful as an execution lens or checklist rather than as a traditional evergreen article.
- PeterOS may need to distinguish between skill libraries worth invoking and durable principles worth promoting into knowledge or reference pages.

## Caveats

- The packaging is Claude-plugin specific, so it is not something to vendor into the wiki as-is.
- The advice is opinionated and book-derived, so it should be treated as a lens rather than a universal operating law.
- The live repo can evolve, so the local note should summarize the role of the resource rather than mirror its full contents.

## Fit For PeterOS

- Best default use: keep it as an external skill-library reference under `reference/ai-agents/skills/`.
- Secondary use: if a specific heuristic from the repo keeps proving useful, promote that heuristic into `reference/company-building/` or `knowledge/company-building/`.
- Weak fit: vendoring the repo into this knowledge base. That would mix tool-distribution concerns into a wiki that is supposed to maintain synthesized knowledge.

## Open Questions

- Which of the 10 skills are actually useful in Peter's workflow versus merely interesting?
- Should company-building skill libraries live only under `ai-agents/skills`, or should especially durable ones also get company-building reference pages?
